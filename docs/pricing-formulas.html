1. 买入价格

实盘策略：买入热路径不先算价，直接用 `swapSimple` exact-in 发送交易。价格在成交后用事件或日志解释。

avg_buy_price = collateral_spent / ot_received

2. 赎回定价

卖出服务用曲线合约对指定 OT 数量报价，链上返回用户到手和金库税两部分。

redeem = calRedeemValueByOtDelta(market, tokenId, otDelta) gross_redeem = collateralToUser + collateralToTreasury

3. 赎回税

赎回税不是从配置里猜，而是由链上报价中的 treasury 部分反推。

protocol_tax_bps = collateralToTreasury / gross_redeem * 10000

4. 磨损与保护

磨损只比较边际现货估值和曲线赎回总值；有效损耗再把协议税和额外卖出税算进去。

spot_value = otDelta * pre_marginal_price / 10^decimals slippage_bps = max(0, spot_value - gross_redeem) / spot_value * 10000 effective_loss_bps = max(0, spot_value - net_to_user) / spot_value * 10000

狙击买入

收到新市场事件后先发 exact-in 买入交易，不让价格分析阻塞成交。

2

异步采样

成交后再读链上价格，记录不同时间点的卖出报价。

3

卖出判断

卖出服务看赎回总值、协议税、额外税和磨损。

4

保护阈值

曲线磨损超过 50% 时不执行卖出，只写日志。
